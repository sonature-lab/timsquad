# Architecture Review Checklist

Architect 3축 분석 시 참조하는 상세 체크 항목.

## Axis 1: 시퀀스 내 태스크 일관성

- 에러 핸들링 패턴이 태스크 간 동일한가
- 네이밍 컨벤션이 통일되어 있는가
- 의존성 방향이 일관된가 (레이어 분리)
- 공통 유틸/헬퍼의 중복 구현이 없는가
- QA 태스크가 developer 태스크의 변경 사항을 실제로 테스트했는가

## Axis 2: SSOT 적합성

- API 엔드포인트의 경로, 메서드, 파라미터가 service-spec과 일치
- 데이터 모델의 필드, 타입, 관계가 data-design과 일치
- 비즈니스 로직이 requirements의 규칙을 충족
- 누락된 엔드포인트/필드/검증이 없는가
- SSOT에 없는 임의 구현이 추가되지 않았는가

## Axis 3: 크로스시퀀스 연속성

- 이전 보고서의 "다음 시퀀스 준비 상태" 조건이 충족되었는가
- 이전 시퀀스에서 확립된 아키텍처 패턴을 따르는가
- 이전 시퀀스의 발견 이슈가 해결되었는가 (해당 시퀀스인 경우)
- 결정 사항 간 모순이 없는가

> 이전 보고서가 없으면 (첫 시퀀스) Axis 3은 건너뜁니다.

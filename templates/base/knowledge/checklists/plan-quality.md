# Plan Quality Checklist

계획 품질 검증 (plan-review 규칙) 시 참조하는 상세 체크 항목.
기존 architecture-review.md의 3축 분석과 동일 형식.

## Axis P1: 요구사항 명확성

- 태스크 설명이 "무엇을"과 "완료 조건"을 모두 포함하는가
- 주요 용어가 glossary.md에 정의되어 있는가
- requirements.md 또는 functional-spec.md에 근거 FR/FS ID가 있는가
- 수용 기준(acceptance criteria)이 테스트로 변환 가능한가
- 스코프 경계가 명확한가 ("이것은 포함, 이것은 미포함")

## Axis P2: 기술 분해 품질

- 태스크가 1 시퀀스(에이전트 3명 이내) 안에 완료 가능한 크기인가
- 시퀀스 내 태스크 간 의존성 순서가 명시되어 있는가
- 각 태스크에 담당 에이전트가 배정되어 있는가
- 데이터 모델 변경이 필요하면 DBA 태스크가 선행하는가
- API 변경이 필요하면 service-spec.md 업데이트가 선행하는가

## Axis P3: 리스크 & 빠진 것

- 기존 코드 중 영향받는 파일/함수가 식별되었는가
- 하위 호환성 깨짐 가능성이 평가되었는가
- 에러 시나리오 대응이 포함되었는가
- 보안 관련 변경이면 security 에이전트 참여가 계획되었는가
- 성능 민감 영역이면 벤치마크 기준이 있는가

> 축 내 5개 항목 중 4개 이상 충족 시 해당 축 PASS.
> 3개 이하면 REVISE 사유에 미충족 항목 명시.

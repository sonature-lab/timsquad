# 실패 패턴 (Failure Patterns)

> 반복적으로 발생하는 실패 패턴을 기록하고 분석합니다.
> 3회 이상 발생 시 자동으로 개선 조치가 제안됩니다.

---

## 패턴 분류

### 카테고리

| 카테고리 | 코드 | 설명 |
|---------|:----:|------|
| 코드 품질 | CQ | 테스트 실패, 타입 에러, 린트 오류 |
| 설계 불일치 | DM | API 불일치, 스키마 불일치, 아키텍처 위반 |
| 요구사항 | RQ | 요구사항 불명확, 스코프 크리프 |
| 프로세스 | PR | Phase 위반, 문서 누락, 워크플로우 오류 |
| 커뮤니케이션 | CM | Handoff 실패, 컨텍스트 손실 |

### FP-001: SSOT 미참조 구현

| 항목 | 내용 |
|-----|------|
| **ID** | FP-001 |
| **카테고리** | DM (설계 불일치) |
| **최초 발견** | - |
| **발생 횟수** | 0회 |
| **영향 에이전트** | tsq-developer |
| **상태** | active |

**증상**
- service-spec.md와 다른 API 응답 형식
- data-design.md와 다른 테이블 구조
- error-codes.md에 없는 에러 코드 사용

**원인 분석**
- 구현 전 SSOT 확인 단계 생략
- SSOT 문서가 최신 상태가 아님
- 에이전트 프롬프트에 SSOT 참조 강제 누락

**영향**
- 프론트엔드-백엔드 불일치
- 테스트 실패
- Level 2 피드백 증가

**개선 조치**
- [ ] validation-rules.xml에 pre-condition 강화
- [ ] tsq-developer SKILL.md에 SSOT 체크리스트 추가
- [ ] 구현 전 SSOT 버전 확인 자동화

### FP-002: 불완전한 Handoff

| 항목 | 내용 |
|-----|------|
| **ID** | FP-002 |
| **카테고리** | CM (커뮤니케이션) |
| **최초 발견** | - |
| **발생 횟수** | 0회 |
| **영향 에이전트** | 전체 |
| **상태** | active |

**증상**
- 다음 에이전트가 컨텍스트를 모름
- 같은 작업 반복 수행
- 결정 사항 누락

**원인 분석**
- workspace.xml handoff 섹션 미작성
- 전달 사항이 너무 간략함
- 참조할 SSOT 문서 누락

**영향**
- 작업 지연
- 토큰 낭비 (컨텍스트 재구성)
- 품질 저하

**개선 조치**
- [ ] handoff 필수 필드 검증 강화
- [ ] handoff 템플릿 개선
- [ ] 자동 handoff 생성 스크립트

### FP-003: Phase 위반

| 항목 | 내용 |
|-----|------|
| **ID** | FP-003 |
| **카테고리** | PR (프로세스) |
| **최초 발견** | - |
| **발생 횟수** | 0회 |
| **영향 에이전트** | 전체 |
| **상태** | active |

**증상**
- Planning phase에서 코드 작성 시도
- SSOT 승인 전 Implementation 시작
- Review 없이 배포 진행

**원인 분석**
- state-machine.xml 규칙 미준수
- Phase 상태 확인 누락
- 급한 마음에 프로세스 생략

**영향**
- 품질 저하
- 재작업 필요
- Level 3 피드백 발생

**개선 조치**
- [ ] Phase 검증 자동화 강화
- [ ] 위반 시 명확한 에러 메시지
- [ ] 위반 로그 기록 및 알림
